# Prompt Engineering Guide: 핵심 기법 총람

> 최종 업데이트: 2026-03 | 출처: https://www.promptingguide.ai/ | https://github.com/dair-ai/Prompt-Engineering-Guide

## 핵심 요약

Zero-Shot부터 Tree of Thoughts까지 LLM 프롬프팅의 주요 기법 14가지를 정리한 레퍼런스. 각 기법의 원리, 예시, 적용 시나리오를 포함하며, 웹소설 자동 생성 파이프라인 관점의 적용 포인트를 제시한다.

---

## 1. Zero-Shot Prompting

**설명**: 예시 없이 모델의 기존 지식만으로 태스크를 수행하게 하는 방식.

**원리**: 대형 언어 모델은 대규모 학습 데이터에서 패턴을 익혔기 때문에, 예시 없이도 명확한 지시만으로 많은 태스크를 처리할 수 있다.

**예시**:
```
감정을 분류하시오: "오늘 날씨가 정말 좋다."
```

**적용 시나리오**:
- 간단한 분류, 번역, 요약 태스크
- 모델이 충분히 학습한 도메인의 명확한 작업
- 빠른 프로토타이핑 단계

**한계**: 복잡하거나 도메인 특화된 작업에서는 성능 저하.

---

## 2. Few-Shot Prompting

**설명**: 소수의 예시(보통 3~5개)를 프롬프트에 포함해 모델이 원하는 출력 패턴을 학습하게 하는 방식.

**원리**: 예시가 모델의 통계적 패턴 분포를 원하는 방향으로 이동시킨다. 구조, 문체, 형식을 동시에 전달할 수 있다.

**예시**:
```
다음 예시를 참고해 감정을 분류하시오.

입력: "서비스가 엉망이었다." → 출력: 부정
입력: "정말 훌륭한 경험이었다." → 출력: 긍정
입력: "그저 그랬다." → 출력: 중립

입력: "기대 이상이었다." → 출력:
```

**적용 시나리오**:
- 특정 형식/문체가 요구되는 창작 작업
- 도메인 특화 분류, 추출 태스크
- 일관성이 중요한 반복 생성 작업

**주의**: 예시는 관련성 높고, 엣지 케이스를 포함하며, 의도치 않은 패턴을 피해야 한다. `<example>` 태그로 감싸면 Claude에서 효과적.

---

## 3. Chain-of-Thought (CoT) Prompting

**설명**: 모델이 최종 답변 전에 중간 추론 단계를 명시적으로 기술하도록 유도하는 방식. Wei et al. (2022) 제안.

**원리**: 단계적 사고 과정을 강제함으로써 복잡한 추론에서 오류율을 낮춘다. 충분히 큰 모델(~100B+)에서 창발적으로 나타나는 능력.

### 변형 기법

**Few-shot CoT**: 추론 과정이 포함된 예시를 제공
```
Q: 5개 사과 중 3개를 먹고 2개를 샀다. 남은 사과는?
A: 처음 5개에서 3개를 먹으면 2개 남는다. 여기에 2개를 사면 4개가 된다. 정답: 4개

Q: 12개 공 중 4개를 잃고 7개를 받았다. 남은 공은?
A:
```

**Zero-shot CoT**: 단순히 "단계별로 생각해보자(Let's think step by step)" 추가
```
Q: 복잡한 수학 문제...
A: 단계별로 생각해보자.
```

**Auto-CoT**: 클러스터링으로 대표 질문을 선택하고 Zero-shot CoT로 체인을 자동 생성 (Zhang et al., 2022)

**적용 시나리오**:
- 수학 문제, 논리 추론, 다단계 의사결정
- 에피소드 플롯 설계 (원인→결과 연쇄 추론)
- 캐릭터 동기 분석, 복잡한 세계관 규칙 적용

---

## 4. Self-Consistency

**설명**: 동일 프롬프트로 여러 추론 경로를 생성하고 다수결로 최종 답을 선택하는 방식. Wang et al. (2022) 제안.

**원리**: 여러 독립적 추론 경로 중 가장 일관된 답변을 채택함으로써 단일 CoT보다 신뢰도를 높인다.

**예시 흐름**:
```
동일 질문 → 추론 경로 A → 답변 X
동일 질문 → 추론 경로 B → 답변 X
동일 질문 → 추론 경로 C → 답변 Y
→ 다수결: 답변 X 채택
```

**적용 시나리오**:
- 정답이 명확히 존재하는 추론 태스크 (수학, 논리)
- 중요한 결정에서 신뢰성 향상
- 아웃라인 검증 시 여러 관점으로 품질 확인

**비용**: 샘플 수만큼 API 호출 증가. 중요한 단계에만 선택적으로 적용 권장.

---

## 5. Tree of Thoughts (ToT)

**설명**: 하나의 선형 추론 경로 대신, 트리 구조로 여러 중간 사고 가지를 동시에 탐색하고 평가·가지치기하는 방식. Yao et al. (2023) 제안.

**원리**: 각 "사고(thought)"는 문제 해결의 중간 단계를 나타내는 일관된 언어 시퀀스. 모델이 사고를 생성하고 동시에 평가한다.

**탐색 알고리즘**:
- BFS (너비 우선): 같은 깊이의 모든 후보를 평가
- DFS (깊이 우선): 유망한 가지를 깊이 탐색
- Beam Search: 상위 K개 후보만 유지

**평가 방식**: 각 중간 사고를 "확실/가능/불가능"으로 분류 후 상위 후보 유지.

**변형**:
- 원본 ToT (Yao et al.): 범용 탐색 전략
- RL 기반 ToT (Long, 2023): 강화학습으로 탐색 전략 자체를 학습
- 단순 프롬프팅 버전: 단일 프롬프트 안에서 여러 전문가 시각 시뮬레이션

**적용 시나리오**:
- 복잡한 플롯 설계 (여러 전개 방향 탐색 후 최선 선택)
- 창의적 작업에서 다양한 아이디어 생성 후 선별
- 전략적 계획이 필요한 장기 작업

---

## 6. ReAct (Reasoning + Acting)

**설명**: 추론과 행동을 교차하며 외부 정보와 상호작용하는 방식. Yao et al. (2022) 제안.

**원리**: Thought(추론) → Action(행동) → Observation(관찰) 사이클을 반복. 외부 도구/API/검색 엔진과 연동해 정보를 동적으로 수집하며 추론.

**사이클 예시**:
```
Thought: 이 캐릭터의 동기를 파악하려면 설정 파일을 확인해야 한다.
Action: read_file("characters.md")
Observation: [파일 내용 반환]
Thought: 설정에 따르면 이 캐릭터는 복수심이 동기다. 이를 에피소드에 반영해야 한다.
Action: write_draft(...)
```

**장점**:
- 환각(hallucination) 감소: 외부 사실 기반 추론
- 투명성 향상: 추론-행동 추적 가능
- 복잡한 멀티스텝 태스크 처리

**적용 시나리오**:
- 에이전트 기반 에피소드 생성 파이프라인
- 로어 파일 참조가 필요한 일관성 검증
- 외부 검색이 필요한 장르 리서치

---

## 7. Retrieval-Augmented Generation (RAG)

**설명**: 외부 지식 베이스에서 관련 문서를 검색해 프롬프트에 추가하고 응답을 생성하는 방식. Lewis et al. (2020) 제안.

**구성 요소**:
- **Parametric Memory**: 모델의 학습된 지식
- **Non-parametric Memory**: 외부 벡터 인덱스 (문서, DB)
- **Retriever**: 관련 문서를 찾는 신경망 검색기

**흐름**:
```
사용자 쿼리 → 검색기 → 관련 문서 추출 → [문서 + 쿼리] → LLM → 응답
```

**장점**:
- 모델 재학습 없이 최신 정보 활용
- 환각 감소 (사실 기반 생성)
- 특정 코퍼스에 특화된 응답

**적용 시나리오**:
- 대규모 로어 파일에서 관련 설정만 동적 추출
- 이전 에피소드 내용 검색 및 일관성 유지
- 장르 레퍼런스 검색 기반 집필

---

## 8. Prompt Chaining

**설명**: 복잡한 태스크를 순차적 프롬프트 체인으로 분해하고 각 단계의 출력을 다음 단계의 입력으로 사용하는 방식.

**패턴**:
```
Stage 1 프롬프트 → 출력 A
Stage 2 프롬프트 [출력 A 포함] → 출력 B
Stage 3 프롬프트 [출력 B 포함] → 최종 결과
```

**적용 시나리오**:
- 프리프로덕션 파이프라인 (장르→구조→아웃라인→플롯)
- 초고 작성→문체 리뷰→수정 체인
- 중간 결과를 검사/분기해야 하는 파이프라인

---

## 9. Generate Knowledge Prompting

**설명**: 답변 전에 모델에게 관련 지식을 먼저 생성하도록 지시하고, 생성된 지식을 추론에 활용하는 방식.

**예시**:
```
먼저 [주제]에 대해 알고 있는 핵심 사실들을 나열하시오.
[생성된 지식 목록]
이제 위 사실을 바탕으로 [질문]에 답하시오.
```

**적용**: 창작 전 세계관 규칙 환기, 장르 관습 점검 후 집필 진행.

---

## 10. Meta Prompting

**설명**: 모델이 자신의 프롬프팅 전략을 스스로 구성하도록 안내하는 방식. 태스크 자체가 아닌 "어떻게 접근할지"를 프롬프트로 제시.

**적용**: 복잡한 창작 태스크 시작 시 모델 스스로 접근 방식을 설계하게 허용.

---

## 11. Reflexion

**설명**: 자기 평가와 반복적 정제를 구현하는 방식. 모델이 출력을 생성하고, 평가하고, 피드백 기반으로 개선.

**사이클**: 생성 → 자기 평가 → 정제 → 반복

**적용**: 초고 작성 후 모델 자체 리뷰 → 수정 루프.

---

## 12. Program-Aided Language Models (PAL)

**설명**: 코드 생성으로 계산 태스크를 처리하는 방식. 수치 계산이나 로직은 코드로 실행.

**적용**: 에피소드 글자 수 검증, 씬 수 카운트 등 정량 검증 자동화.

---

## 13. Automatic Reasoning and Tool-use (ART)

**설명**: 추론과 외부 도구 사용을 자동으로 결합하는 방식. 태스크 라이브러리에서 관련 예시를 찾아 멀티스텝 추론과 도구 호출을 조합.

**적용**: 파이프라인 에이전트가 도구(파일 읽기, Python 검증, 외부 API)를 자동 조합하는 설계.

---

## 14. Multishot / Structured Examples

**설명**: 3~5개 예시를 `<example>` 태그로 구조화해 제공하는 방식 (Claude 권장).

**예시 구조**:
```xml
<examples>
  <example>
    <input>입력 텍스트 1</input>
    <output>원하는 출력 1</output>
  </example>
  <example>
    <input>입력 텍스트 2</input>
    <output>원하는 출력 2</output>
  </example>
</examples>
```

## 우리 파이프라인 적용 포인트

| 파이프라인 단계 | 권장 기법 | 이유 |
|---|---|---|
| Stage 1 장르 리서치 | ReAct + RAG | 외부 검색 + 문서 기반 추론 |
| Stage 2 구조 설계 | CoT + Few-Shot | 단계적 서사 추론, 예시 기반 구조화 |
| Stage 3 아웃라인 생성 | Prompt Chaining + ToT | 전 EP 연결 추론, 여러 방향 탐색 |
| Stage 4 플롯 설계 | CoT + Self-Consistency | 클리프행어 연결 검증 |
| Stage 5 아웃라인 검증 | Reflexion + Self-Consistency | 자기 평가 + 다수결 검증 |
| Stage 2 (프로덕션) 초고 | Few-Shot + Generate Knowledge | 문체 예시 + 로어 환기 |
| Stage 3/4 리뷰 | Reflexion + Parallel | 문체/일관성 동시 검토 |
| Stage 6 검증 | PAL (Python) | 정량 검증 자동화 |
