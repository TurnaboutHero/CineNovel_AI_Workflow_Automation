# 도메인 전문가 프라이밍 기법 가이드

> 최종 업데이트: 2026-03 | 출처:

> - https://arxiv.org/abs/2602.02752 (Domain Knowledge Strategies for LLM Optimization)

## 핵심 요약

도메인 프라이밍은 LLM이 학습 데이터에서 특정 도메인의 패턴, 어휘, 추론 방식을 활성화하도록 시스템 프롬프트를 설계하는 기법이다. 역할 부여, 도메인 어휘 주입, 예시 기반 패턴 활성화, 컨텍스트 사전 로딩 등을 조합해 모델을 특정 도메인 전문가처럼 행동하게 만든다.

---

## 1. 패턴 프라이밍의 원리

### 통계적 분포 이동

LLM은 명령을 "실행"하는 것이 아니라 학습 데이터의 통계적 패턴을 기반으로 토큰을 예측한다. 프롬프트의 언어와 구조가 모델이 "어떤 종류의 텍스트" 공간에서 작동하는지를 결정한다.

**핵심 통찰**: "역할('CISO로서'), 어조('유머러스하게'), 독자('고등학생 대상')를 언급하면 LLM이 학습 코퍼스에서 관련 패턴을 검색하도록 유도한다. 내용뿐 아니라 밀도와 어조에서도."

### Descriptor Stacking (설명자 중첩)

관련 형용사를 조합하면 영향력이 증폭된다:

```
# 약한 프라이밍
정확하게 답하세요.

# 강한 프라이밍 (Descriptor Stacking)
객관적이고, 비판적이며, 상세하고, 사실에 기반한, 기술적으로 정확한 방식으로 답하세요.
```

이 중첩은 과학적/기술적 글쓰기에서 해당 표현이 자연스럽게 공존하는 형식적, 분석적 패턴을 활성화한다.

## 2. 핵심 프라이밍 기법 6가지

### 기법 1: 컨텍스트 프라이밍 (Context Priming)

프롬프트를 도메인 맥락을 확립하는 실질적 단락으로 시작. "LLM에게 미니 강의"를 제공해 관련 지식 기반으로 주의를 향하게 한다.

**구조**:
```
[도메인 컨텍스트 단락]
[핵심 도메인 개념과 관계]
[현재 태스크]
```

**한국 웹소설 도메인 예시**:
```
한국 웹소설은 주로 카카오페이지, 네이버 웹소설 등의 플랫폼에서 연재되는 장편 소설입니다.
로맨스 판타지 장르는 현대 여성 독자를 주 대상으로, 이세계 전생이나 빙의 설정에서
강력하고 매력적인 남주인공과의 로맨스를 중심으로 전개됩니다. 각 화는 3,000~6,000자로
구성되며, 화 말미의 클리프행어(cliff-hanger)가 독자 유지율의 핵심입니다. 독자들은
감정선의 세밀한 묘사, 심리 묘사, 그리고 "떡밥 회수"(이전에 암시된 복선의 해결)에
높은 가치를 둡니다.
```

### 기법 2: 키워드 시딩 (Keyword Seeding)

도메인 특화 용어를 컨텍스트 서문에 전략적으로 배치해 모델의 주의를 특수 지식 영역으로 유도:

```
다음 설화 세계관에서의 위계는 다음과 같습니다: 귀족 > 평민 > 천민.
마나(mana)는 마력의 원천으로, 혈통에 따라 초기량이 결정됩니다.
계약 마법(Contract Magic)은 양 당사자의 합의로만 해제 가능한 구속력 있는 서약입니다.

위 세계관의 규칙을 엄격히 따르면서 EP-07을 작성하세요.
```

**효과**: 시딩된 키워드들이 프롬프트 전반의 주의를 해당 도메인 패턴으로 집중시킨다.

### 기법 3: 퓨샷 도메인 예시 (Few-Shot Domain Examples)

도메인 적용을 보여주는 구체적 케이스 예시 제공. 단순히 패턴을 "보여주는" 것이 아니라 도메인 추론 방식을 활성화한다.

```xml
<examples>
  <example>
    <scenario>주인공이 배신당하는 장면</scenario>
    <good_writing>
      심장이 차갑게 식어갔다. '내가 잘못 본 건가.' 아르테미는 세 번이나 눈을 깜빡였지만,
      레오나르도의 손이 세실리아의 허리를 감싸고 있다는 사실은 달라지지 않았다.
      웃음소리가 들렸다. 그것이 자신이 내뱉는 것임을 깨달은 건 주변 사람들이 돌아보고 나서였다.
    </good_writing>
    <key_elements>내면 심리 + 감각 묘사 + 부정과 수용의 단계 + 반응의 지연</key_elements>
  </example>
  <example>
    <scenario>클리프행어 화 말미</scenario>
    <good_writing>
      "그분이 오셨습니다."
      집사의 목소리가 떨렸다. 아르테미가 고개를 들었을 때, 서재 문이 천천히 열렸다.
      그리고 그 뒤에 서 있는 사람을 본 순간—
      [다음 화에 계속]
    </good_writing>
    <key_elements>예상치 못한 인물 등장 + 감정 정점 + 강제 중단 + 독자 궁금증 극대화</key_elements>
  </example>
</examples>
```

### 기법 4: 단계별 추론 스캐폴딩 (Step-by-Step Reasoning Scaffolding)

도메인 프레임워크 안에서 지식 검색, 합성, 의사결정을 안내하는 번호 매긴 단계 구조:

```
에피소드 작성 전 다음 순서로 진행하세요:

1단계 [지식 검색]: characters.md에서 이 화 등장 캐릭터의 현재 상태(감정, 목표, 관계)를 확인하세요.
2단계 [아웃라인 분석]: 이 화의 씬 비트를 검토하고 각 씬의 목적을 파악하세요.
3단계 [이전 화 연결]: EP-06 마지막 장면에서 자연스럽게 이어지는 시작점을 결정하세요.
4단계 [집필]: 1~3단계의 분석을 바탕으로 초고를 작성하세요.
5단계 [자기 검증]: 클리프행어가 충분히 강렬한지, 캐릭터 일관성이 유지되는지 확인하세요.
```

### 기법 5: 휴리스틱 유도 (Heuristic Elicitation)

직접 휴리스틱을 요청하는 대신, 예시 케이스 안에 도메인 규칙(경험칙)을 내장시켜 모델이 추출하고 적용하게 한다:

```xml
<domain_heuristics>
  다음 원칙들은 한국 웹소설 독자 반응 분석에서 도출된 경험칙입니다:
  - 감정 씬은 대화보다 내면 묘사가 더 많아야 한다 (7:3 비율)
  - 첫 만남 장면에서 남주인공은 말보다 행동으로 매력을 드러내야 한다
  - 오해 장면은 독자가 "직접 설명하면 해결될 텐데"라고 느끼게 만들어야 최고 반응이 나온다
  - 클리프행어는 새로운 질문을 만들어야지, 기존 질문의 답을 주면 안 된다
</domain_heuristics>
```

### 기법 6: 검색 보강 프롬프팅 (Retrieval-Augmented Prompting)

도메인 특화 데이터베이스를 유지하고 모델에게 먼저 검색 후 응답하도록 지시:

```
먼저 아래 설정 데이터베이스에서 다음 정보를 찾으세요:
1. 이 씬에 등장하는 모든 캐릭터의 현재 감정 상태
2. 현재 씬의 장소에 관련된 세계관 규칙
3. 이 화에서 회수되어야 할 이전 복선

[설정 데이터베이스]
...

위 정보를 종합해 에피소드를 작성하세요.
```

## 3. 역할 부여 (Role Injection) 심화

### 레이어드 역할 설계

단순한 "당신은 작가입니다" 대신, 다층적 역할을 부여:

```
당신은 다음과 같은 전문가입니다:
- 1차 정체성: 한국 웹소설 베스트셀러 작가 (10년 경력, 누적 조회수 1억 이상)
- 전문 분야: 로맨스 판타지, 특히 이세계 빙의물
- 문체 특성: 간결하면서도 감정이 풍부, 속도감 있는 전개, 강렬한 클리프행어
- 독자 이해: 20-35세 여성 독자의 취향과 감정선 패턴을 깊이 이해
- 작업 방식: 아웃라인을 철저히 따르되 현장에서 유연하게 조율
```

### 조건부 역할 전환

```
기본적으로 집필 에이전트로 행동하세요.
검증 단계에서는 편집장 역할로 전환해 비판적으로 검토하세요.
이미지 프롬프트 작성 시에는 시각적 디렉터 역할로 전환하세요.
```

## 4. 도메인별 패턴 활성화 체크리스트

### 한국 웹소설 도메인 체크리스트

**어휘 및 표현**:
- [ ] 장르 특화 용어 사용 (빙의물, 회귀물, 역하렘, 먼치킨, OP 주인공 등)
- [ ] 한국 웹소설 특유의 플롯 패턴 언급 (서브컬처 맥락 활성화)
- [ ] 플랫폼별 특성 언급 (카카오페이지 vs 네이버웹소설 독자 차이)

**서사 구조 패턴**:
- [ ] 화별 구조 명시 (도입→전개→클라이맥스→클리프행어)
- [ ] 복선 회수 시점 표시
- [ ] 캐릭터 성장 호 단계 명시

**독자 반응 패턴**:
- [ ] "독자 이탈 포인트" 사전 식별 및 방지 지시
- [ ] 감정 피크 씬 위치 명시
- [ ] "포인트 소비 가치" 체크 (유료 콘텐츠 기준)

## 5. 도메인 프라이밍 강도 조절

| 상황 | 프라이밍 강도 | 방법 |
|---|---|---|
| 일반적 집필 태스크 | 중간 | 역할 + 키워드 시딩 |
| 장르 특화 씬 (감정 절정) | 높음 | 역할 + 예시 + 휴리스틱 |
| 새 에이전트/세션 시작 | 높음 | 전체 컨텍스트 프라이밍 |
| 연속 집필 (중간 화) | 낮음 | 역할만 (이미 패턴 활성화) |
| 검증 태스크 | 다른 역할 | 편집자/독자 시각 전환 |

## 6. 안티패턴 (피해야 할 것들)

**너무 일반적인 역할 부여**:
```
# 나쁜 예
당신은 작가입니다.

# 좋은 예
당신은 로맨스 판타지 전문 한국 웹소설 작가입니다.
독자 이탈률을 최소화하는 강렬한 클리프행어 기법에 특히 능합니다.
```

**도메인 어휘 없는 지시**:
```
# 나쁜 예
긴장감 있게 써주세요.

# 좋은 예
이 씬은 소이 반전(독자가 예상한 클리셰가 뒤집히는 순간)의 직전입니다.
긴장감을 최대로 끌어올린 후 독자의 예상을 완전히 뒤집으세요.
```

**예시 없는 추상적 지시**:
```
# 나쁜 예
한국 웹소설 특유의 문체로 써주세요.

# 좋은 예
[위의 few-shot domain examples 방식으로 구체적 예시 3개 제공]
```

## 7. 파인튜닝 vs 프롬프팅 결정 기준

| 조건 | 권장 접근 |
|---|---|
| 도메인 데이터 < 1,000 예시 | 프롬프팅 (도메인 프라이밍) |
| 도메인 데이터 > 10,000 예시 | 파인튜닝 고려 |
| 빠른 프로토타이핑 필요 | 프롬프팅 |
| 고정된 반복 태스크, 비용 최적화 | 파인튜닝 |
| 도메인이 자주 변경 | 프롬프팅 (유연성) |
| 최고 수준의 도메인 특화 필요 | RAG + 프롬프팅 + 파인튜닝 조합 |

## 우리 파이프라인 적용 포인트

### Stage 2 초고 집필 - 도메인 프라이밍 템플릿

```xml
<system>
당신은 한국 웹소설 로맨스 판타지 전문 작가입니다.
누적 조회수 5,000만 이상의 베스트셀러 작품 경험 보유.
독자 유지율과 클리프행어 효과에 특히 능합니다.

<domain_context>
설화는 귀족 사회를 배경으로 한 이세계 빙의물입니다.
현대 여성(아르테미)이 소설 속 악역으로 빙의해 자신의 사망 플래그를 피하는 이야기.
독자층: 20-35세 여성, 로맨스 감정선과 두뇌 플레이에 높은 가치를 둠.
장르 관습: 먼치킨 요소 있으나 주인공은 현대 지식을 활용한 지략으로 극복.
</domain_context>

<style_priming>
객관적, 감정이입적, 속도감 있는, 심리 묘사가 풍부한 문체.
감정 씬: 내면 묘사 70% + 대화/행동 30%.
클리프행어: 새로운 질문 생성 (기존 질문 답변 금지).
</style_priming>
</system>
```

### Stage 5 아웃라인 검증 - 도메인 전문가 역할 전환

```xml
<system>
당신은 한국 웹소설 플랫폼의 수석 편집장입니다.
독자 이탈률 분석, 화별 완독률 최적화, 장기 연재 플롯 관리 전문.
비판적이고 독자 관점에서 아웃라인을 평가합니다.

평가 기준:
1. 각 화의 클리프행어 강도 (독자가 다음 화를 결제할 동기)
2. 복선의 균형 (너무 많으면 혼란, 너무 적으면 허무)
3. 캐릭터 성장 호의 자연스러움
4. 시즌 중반 슬럼프 방지 (20화~30화 구간 특히 위험)
</system>
```
